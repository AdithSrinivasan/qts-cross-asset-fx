# qts-cross-asset-fx

Winter 2026 FINM 33150 Quantitative Trading Strategies — Cross-Asset FX Futures Trading Project

Research project for the FINM 33150 Quantitative Trading Strategies course, in fulfillment of the MS Financial Mathematics at the University of Chicago. The objective is to study whether lead–lag relationships between **equity and credit markets** can predict **FX futures returns**, and to evaluate tradable strategies under realistic trading assumptions.

## Project Overview
This repository contains a research and backtesting framework for testing cross-asset predictability using equity (and credit proxies where relevant) as signals for FX futures. The strategy focuses on a set of developed and emerging markets with varying liquidity, using standardized FX futures for implementation. Evaluation is conducted via walk-forward backtesting with transaction costs and contract rolls.

**Core questions**
- Do equity/credit signals lead FX futures in select markets?
- Are effects stronger in less liquid or higher FX-demand markets?
- Are signals robust out-of-sample and across market regimes?

## Setup

### 1. Create the conda environment

```bash
conda create -n qts-fx-v2 python=3.12 -y
conda activate qts-fx-v2
pip install pandas numpy polars plotnine matplotlib statsmodels scikit-learn linearmodels pyhdfe openpyxl yfinance scipy jupyter
```

### 2. Extract futures data

The 1-minute FX futures data is stored as a zip archive. Extract it before running the backtester:

```bash
cd data
unzip futures_1m.zip
cd ..
```

This creates the `data/futures_1m/` directory with subdirectories per country (Australia, Brazil, Canada, etc.).

## Running the Analysis & Backtest (End-to-End)

The full pipeline has three steps: (1) generate data and signals via the analysis notebook, (2) build the futures panel, and (3) run the backtester.

### Step 1: Run the analysis notebook to generate all data files

The analysis notebook (`notebooks/analysis-updated.ipynb`) is the primary entry point. It generates all intermediate data files needed by the backtester.

```bash
conda activate qts-fx-v2
cd notebooks
jupyter notebook analysis-updated.ipynb
```

Run all cells in order. This will:
- Load and prepare FX spot data from FRED CSV files (`data/fx_data/`)
- Compute FX excess returns and daily carry
- Run Stage 1 panel regressions (hedging out carry and dollar factor)
- Save Stage 1 residuals to `data/stage1_residuals.csv`
- Run Stage 2 OLS and Random Forest models
- Save predictions to `data/rf_train_predictions.csv` and `data/rf_test_predictions.csv`
- Generate entry/exit thresholds to `data/rf_thresholds.csv` and `data/rf_exit_thresholds.csv`
- Save hedge ratios to `data/is_hedge_ratios.csv`

### Step 2: Build the FX futures panel

After extracting `futures_1m.zip` (see Setup), build the after-close futures panel:

```bash
conda activate qts-fx-v2
python -c "from src.load_data import load_fx_futures_data, build_after_close_panel; build_after_close_panel(load_fx_futures_data())"
```

This generates `data/fx_futures_panel.csv`.

### Step 3: Run the backtester

```bash
conda activate qts-fx-v2
cd backtester
python main.py
```

This runs the full backtest using:
- Signal predictions from Step 1
- Entry/exit thresholds from Step 1
- FX futures panel from Step 2
- Contract specs from `data/fx_contract_specs.csv`

The backtester outputs portfolio statistics (Sharpe ratio, max drawdown, cumulative returns) and generates performance plots.

Alternatively, the backtester can be run interactively via the notebook:

```bash
cd backtester
jupyter notebook tester.ipynb
```

## Methodology
- **Signals (X):** equity index returns, factor returns, volatility/regime features; optional credit proxies (e.g., CDS where available).
- **Targets (Y):** next-period FX futures returns (excess returns where appropriate).
- **Models:** linear baselines (OLS), with Random Forest for signal generation.
- **Portfolio:** cross-sectional ranking on FX futures; risk controls; transaction costs; futures rolls.
- **Validation:** rolling walk-forward backtests; IC by year; regime analysis.

## Notes / Risks
- Strict timing enforced to avoid lookahead (equity information at _t_ mapped to FX returns at _t+1_).
- FX liquidity differences are expected to attenuate signal strength in the most liquid pairs.
- Overfitting controlled via walk-forward validation and simple baselines prior to complex models.
